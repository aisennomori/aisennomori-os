# DATA_AUDIT.md

このドキュメントは、月読みカードのデータで実際に発生した不整合と、その修正記録を残す。
`CHANGELOG.md` が「何を変えたか」の記録であるのに対し、
このファイルは「データがどう壊れ、どう検証し、どう直したか」というプロセスの記録である。

画像追加やデータ投入は今後も発生する作業のため、次に同種の作業をする際のチェックリストとして機能させる。

各インシデントは、以下の項目で統一して記録する。

| 項目 | 内容 |
|---|---|
| 発生の経緯 | どう発覚したか |
| 内容 | 何がどうズレていたか |
| 原因 | なぜ起きたか |
| 対応 | どう直したか |
| 教訓 | 何を学んだか |
| 再発防止策 | 次に同種の作業をするとき、具体的に何をすべきか |

## Audit 001：11〜20夜の並び順の不一致

| 項目 | 内容 |
|---|---|
| 発生の経緯 | Goh様から「元素と神様」情報を追加したいと、新しい参照テーブル（30夜分の英語名・カナ・元素・神様のリスト）が共有された。このリストと、それまで使っていたエクセルの並び順を突き合わせたところ、11〜20夜で名前がズレていることが判明した |
| 内容 | 11=Laʻau-pau→Huna、12=Huna→Mōhalu、13=Mohalu→Hua、14=Hua→Akua、15=Akua→Hoku、16=Hoku→Māhealani、17=Mahealani→Kulu、18=Kulu→Lāʻau-kū-kahi、19=Laʻau-ku-kahi→Lāʻau-kū-lua、20=Laʻau-ku-lua→Lāʻau-pau（すべて番号がズレていた） |
| 原因 | 旧エクセルと新しい参照テーブルとで、11〜20夜の並び順が異なっていた（どちらかが誤りというより、資料のバージョン違い） |
| 対応 | Goh様が確認・修正した最新のエクセルを再アップロードしてもらい、それを正として全文（行動〜ラッキーカラー）を再取得。画像ファイルも番号ではなく名前を基準に再配置。着手前に影響範囲（テキストと画像の両方）をGoh様に明示し、判断を仰いだ |
| 教訓 | 複数のデータソース（エクセル・参照テーブル・PPT）を扱う際は、「番号」ではなく「名前」を主キーとして突き合わせること |
| 再発防止策 | 21〜30夜のデータ・画像を追加する際は、番号だけで一致確認せず、必ず夜の名前（英語名）とmoodの説明文の両方を突き合わせてから確定させる |

## Audit 002：20日目（La'au-pau）のフィールドずれ

| 項目 | 内容 |
|---|---|
| 発生の経緯 | Goh様がPowerPointのスクリーンショットから正しいテキストを再度書き出し、アプリの表示内容と突き合わせたところ、20日目のカードで項目の中身が入れ替わっていることが発覚した |
| 内容 | 「行動」欄に本来「人間関係」の内容が、「特徴」欄に本来「職業」の内容が、「健康」欄に本来「ラッキーカラー」の内容が入っていた |
| 原因 | Audit 001の並び順修正作業を行った際、フィールドのマッピング処理に誤りがあり、Claude自身の処理過程で発生した。コパイロット側の作業ミスではなかった |
| 対応 | Goh様がスクリーンショットから書き起こした一次情報を正として、全フィールドを再構築した |
| 教訓 | データの再構築・並び替えを自動処理で行う際は、並び順が合っていても、フィールドの対応がずれるケースがある |
| 再発防止策 | データを一括で再マッピングした後は、必ず1件（できれば内容が複雑な夜）を選んでフィールド単位でサンプル検証を行ってから確定する |

## Audit 003：1〜10夜の反映漏れ

| 項目 | 内容 |
|---|---|
| 発生の経緯 | Goh様から「1日目の②の情報が欠落している」との指摘を受け調査したところ、11〜20夜は正しくアプリに反映されていたが、1〜10夜はテキストとして書き出しただけで、アプリ本体には未反映のままだったことが判明した |
| 内容 | 1〜10夜のカードが、①のサブタイトルのみを持つ旧データのまま、アプリに残っていた |
| 原因 | 11〜20夜の反映依頼と、1〜10夜のテキスト化依頼が別々のタイミングで行われ、1〜10夜側に「アプリへ反映する」という明示的な指示がなかったため、テキスト提示のみで作業が止まっていた |
| 対応 | 1〜10夜・11〜20夜の両方について、①②のサブタイトルを両方とも保持する形にデータ構造を見直した上で、まとめて反映した |
| 教訓 | 複数バッチに分けて進める作業では、「テキスト化」と「アプリへの反映」を別の作業として区別する必要がある |
| 再発防止策 | 「テキスト化しました」と「アプリに反映しました」を毎回明確に区別して報告し、反映が済んでいない範囲がある場合は都度明示する |
