# SCREEN_SPEC.md

## 画面構成

月読みカードは、1画面・1カード表示のシングルビューアプリです。
モバイル（スマートフォン）での利用を第一に設計し、デスクトップでも使えるようにしています。

```
┌─────────────────────────────┐
│  月読みカード・愛泉の杜   🌑 01/30 │  ← トップバー
├─────────────────────────────┤
│ ←            →              │
│  ┌─────────────────────┐    │
│  │ [画像]      DAY 01   │    │  ← カードヘッダー
│  │             Hilo ヒロ│    │
│  │             月の先駆者／...│
│  │             古来ハワイ太陰暦1日目│
│  │             🌬風  ...  │    │
│  ├─────────────────────┤    │
│  │ 行動    │ 特徴        │    │  ← 2列パズルタイル
│  │ 健康    │ 人間関係     │    │
│  │ 職業    │ ラッキーカラー│    │
│  └─────────────────────┘    │
├─────────────────────────────┤
│         ● ● ● ● ●            │  ← ドットインジケーター
│   ← スワイプ／矢印キーで移動 →   │
└─────────────────────────────┘
```

## トップバー

- 左：ブランドラベル「月読みカード・愛泉の杜」
- 右：月齢グリフ（実際の月相絵文字）＋「◯◯ / 30」カウンター

## カードヘッダー

- 右上：サムネイル画像（76×76px、角丸）。`hasImage=false` の場合は準備中アイコンを表示
- 左：DAYタグ／英語名＋カナ／meaning（① ／ ②）／calendar／元素・神様タグ

## データ本体（6項目タイル）

- 2列グリッドのタイル形式（行動・特徴・健康・人間関係・職業・ラッキーカラー）
- 各タイルは箇条書き（`\n` 区切りを `・` マーカー付きリストに変換）
- `action` が空の場合、6項目タイルの代わりに
  「この夜の詳しい物語（行動・特徴・健康など）は、近日公開予定です。」という
  控えめな注記ボックスを1つ表示する（`detail-pending`）

## ナビゲーション

| 操作 | 挙動 |
|---|---|
| 左右の矢印ボタン（`prevBtn` / `nextBtn`） | 前後の夜へ移動 |
| スワイプ（タッチ） | 横方向の移動量が50px超、かつ縦移動より大きい場合に発火 |
| キーボード `←` `→` | 前後の夜へ移動（デスクトップ） |
| 下部ドットのクリック | 該当の夜へ直接ジャンプ |

- 30夜は循環する（30日目の次は1日目に戻る、1日目の前は30日目）
- ドットは30個すべてを常に表示せず、現在位置を中心にウィンドウ表示（9個）してUIの圧迫を避けている

## 状態のパターン

| 状態 | 条件 | 表示 |
|---|---|---|
| 完成夜 | `name_en` あり／`action` あり | 通常のフルカード |
| 詳細未着手 | `name_en` あり／`action` なし | ヘッダーのみ＋「近日公開予定」注記（21〜30夜が該当） |
| 完全未着手（フォールバック） | `name_en` すらない | 独立した準備中カード（現状は使用されていないが、将来のデータ欠損に対する安全策として実装している） |
| 画像未着手 | `hasImage=false` | サムネイル部分にアイコンのみのプレースホルダー |

## モーション

- カード送りは `translateX` によるスライドアニメーション（0.45秒、`cubic-bezier(.65,.05,.36,1)`）
- `prefers-reduced-motion` が有効な環境では、トランジションを無効化する
