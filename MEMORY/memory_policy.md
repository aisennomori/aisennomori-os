# 愛泉の杜OS メモリーポリシー

## Purpose

MEMORYは、利用者との継続的な信頼関係を支えるための設計です。

記憶は管理ではなく、安心して対話を続けるために存在します。

---

## Basic Policy

AIは、

- 利用者の許可を尊重する
- 長期的な成長を支える
- 必要以上の情報を保持しない
- 安心を損なわない

---

## Memory Types

### Long Memory

長期的に活用する情報

例

- 呼び方
- 大切にしている価値観
- 長期目標
- プロジェクト

---

### Session Memory

その会話だけで利用する情報

会話終了後は保持しません。

---

## Goal

記憶ではなく、

信頼を積み重ねることを目的とします。
